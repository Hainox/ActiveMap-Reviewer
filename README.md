# ActiveMap Reviewer — Префектура САО

> **Программа для массовой проверки заданий ActiveMap.**
> Открывает задания одно за другим, показывает фотографии и поля, позволяет одним нажатием изменить статус прямо в системе.

## Что это и зачем

При проверке сотен заданий в ActiveMap приходится открывать каждое вручную, смотреть фото, переключать статус — это очень долго.

**ActiveMap Reviewer** делает это в 3 клика:

```
Запустил → Выбрал фильтры → Нажимаешь кнопки решений — статусы меняются автоматически
```

**Возможности:**

- Загружает задания по выбранному виду работ, этапу и стадии
- Список организаций загружается полностью (все записи), отсортирован по алфавиту
- Показывает фотографии (с увеличением), описание, адрес, исполнителя, кастомные поля
- Кнопки решений настраиваются: каждая несёт **Этап** и **Стадию** — оба значения уходят в систему одним нажатием
- Горячие клавиши `1`, `2`, `3`... для скоростной работы
- Несколько сотрудников могут работать параллельно — каждый видит актуальный список
- Работает полностью в браузере, данные идут напрямую на сервер `sao.geofsm.ru`

## Как запустить

### Способ 1 — готовая программа `.exe` ✅ (рекомендуется)

> Не требует Python, ничего устанавливать не нужно.

1. Перейдите в раздел **[Releases](../../releases)** (справа на странице)
2. Скачайте файл `ActiveMapReviewer.exe`
3. Положите его в любую папку и запустите двойным кликом
4. Браузер откроется автоматически на адресе `http://localhost:8765`

> ⚠️ **Если Windows предупреждает об опасности** — нажмите «Подробнее» → «Выполнить всё равно».
> Программа не содержит вирусов, предупреждение появляется для любого нового .exe без подписи.

---

### Способ 2 — запуск через Python

> Для тех, у кого уже установлен Python 3.8+.

1. Скачайте `reviewer.py` и `reviewer_html.html` в **одну папку**
2. Откройте командную строку (cmd) в этой папке
3. Выполните:

   ```
   python reviewer.py
   ```

4. Браузер откроется автоматически

## Экран настройки

После входа выберите параметры проверки:

| Поле | Что выбрать |
|------|-------------|
| **Вид задания** | Тип работ, например «АБП и Бортовой камень (УМиК)» |
| **Статус** | Нужный статус заданий, например «На утверждении» |
| **Стадия** | «В работе», «Завершено» или «Все стадии» |
| **Организация** | Подрядчик (опционально — для фильтрации) |
| **Период** | Дата от/до (опционально) |
| **Кнопки решений** | Настройте от 1 до 5 кнопок — каждая содержит Этап и Стадию |

### Как настроить кнопки решений

Каждая кнопка — это действие, которое вы будете применять к заданиям:

- **Название кнопки** — как она будет выглядеть (например: «✅ Утверждено», «🔄 Доработка»)
- **Этап** — на какой статус переключить задание в системе при нажатии
- **Стадия** — «В работе» или «Завершено» (применяется одновременно с Этапом)

Пример настройки:

| Кнопка | Этап | Стадия |
|--------|------|--------|
| ✅ Утверждено | Утверждено | Завершено |
| 🔄 Доработка | На доработку | В работе |

Нажмите **«Начать проверку»** — загрузится список заданий.

## Просмотр заданий

```
┌─────────────────────────────────────────────────────┐
│  Задание 12 / 91  #5814987                          │
│  ДТ\Фестивальная ул. 52, 52 к.1                     │
│  [АБП и Бортовой камень]  [В работе]  [16.06.2026]  │
├──────────────────────┬──────────────────────────────┤
│                      │  Район: Головинский           │
│   [  Фото  ]         │  Привязка: ДТ                 │
│                      │  Исполнитель: ...             │
│                      │                               │
│                      │  РЕШЕНИЕ:                     │
│                      │  [ 1. Утверждено ]            │
│                      │  [ 2. Доработка  ]            │
│                      │  [ ⏭ Пропустить ]             │
├──────────────────────┴──────────────────────────────┤
│      ← Назад               Вперёд →                 │
└─────────────────────────────────────────────────────┘
```

**Работа с фотографиями:**

- Нажмите на миниатюру — откроется увеличенный просмотр
- Прокрутка мышью — зум (от 100% до 800%)
- Перетаскивание зажатой кнопкой мыши — панорамирование
- `+` / `-` на клавиатуре — зум
- `Escape` — закрыть фото

**Принятие решения:**

- Нажмите нужную кнопку или клавишу `1`, `2`, `3`...
- Этап и Стадия **автоматически** изменятся в системе ActiveMap
- Программа сразу перейдёт к следующему заданию

**Навигация:**

- `←` / `→` — предыдущее/следующее задание
- `S` или `Ы` — пропустить (без изменения статуса)

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `1` — `9` | Кнопка решения № + автопереход к следующему заданию |
| `S` / `Ы` | Пропустить задание (без изменения статуса) |
| `←` | Предыдущее задание |
| `→` | Следующее задание |
| `+` / `=` | Увеличить фото (в лайтбоксе) |
| `-` | Уменьшить фото (в лайтбоксе) |
| `0` | Сбросить зум фото |
| `Escape` | Закрыть фото |
| `Enter` | Войти (на экране входа) |

## Частые вопросы

**❓ Программа не открывается / браузер не запускается**

Откройте браузер вручную и перейдите по адресу: `http://localhost:8765`

**❓ Написано «Задания не найдены»**

Проверьте фильтры — возможно, выбранная комбинация вида работ + статус + стадия не даёт результатов. Попробуйте выбрать «Все стадии».

**❓ Загрузилось ровно 500 заданий с предупреждением**

Система ограничивает выдачу 500 заданиями за раз. Используйте фильтры по периоду (дата от/до) или по организации, чтобы сузить выборку.

**❓ Статус не поменялся / ошибка отправки**

Проверьте подключение к сети. Нажмите кнопку решения ещё раз — программа повторит отправку.

**❓ Фотографии не загружаются**

Иногда сервер отвечает медленно. Подождите несколько секунд или перейдите к следующему заданию и вернитесь обратно.

**❓ Несколько сотрудников работают с одинаковыми заданиями**

Так и задумано — каждый видит один и тот же список, но статусы меняются в системе сразу. Используйте фильтр по организации, чтобы разделить зоны ответственности.

**❓ Можно ли вернуться к заданию, которое уже прошли?**

Да. Нажмите **«История»** в верхней панели — там список всех решений текущей сессии, можно кликнуть на любое и изменить решение.

## Сообщить об ошибке

Нашли баг или что-то работает неправильно?

1. Нажмите кнопку **🐛 Ошибка** в правом верхнем углу программы
2. Опишите что произошло
3. Нажмите «Отправить»

Отчёт сохранится в файл `bug_reports.jsonl` рядом с программой.

Также можно [создать Issue](../../issues) на GitHub.
