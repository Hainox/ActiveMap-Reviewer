# CHANGELOG

Все значимые изменения фиксируются здесь.

---

## [1.2.0] — 2026-06-22

### Добавлено

- Кнопки решений теперь содержат оба поля: **Этап** (статус) и **Стадия** (В работе / Завершено)
- При нажатии кнопки решения в ActiveMap устанавливается `statusId` и `stage` одновременно
- Список организаций: полная загрузка через пагинацию (все страницы по 500 записей), сортировка по алфавиту

### Изменено

- Стадия карточки в фильтре настройки теперь обязательна (нет автовыбора «В работе»)

---

## [1.1.0] — 2026-06-16

### Добавлено

- Новый профессиональный дизайн: тёмный топбар с золотым акцентом (`#c8a96a`), брендинг Префектуры САО
- Экран входа с двухпанельным макетом (тёмная левая панель, белая правая)
- Карточка задания в новом стиле с улучшенной читаемостью

### Исправлено

- Лайтбокс не открывался при клике на фото: `JSON.stringify` с двойными кавычками ломал HTML-атрибут `onclick`

---

## [1.0.0] — 2026-06-15

Первый стабильный релиз.

### Добавлено

- Авторизация по логину/паролю ActiveMap (автодетект формата токена)
- Фильтрация заданий: вид работы, этап (статус), стадия (В работе / Завершено)
- Фильтр по организации/подрядчику
- Фильтр по периоду (дата от/до)
- Просмотр фотографий с лайтбоксом (до 15 фото)
- Кастомные поля задания с читаемыми названиями
- Настраиваемые кнопки решений (до 5 кнопок) с привязкой к статусам
- Горячие клавиши: `1`–`9` (решение + автопереход), `S`/`Ы` (пропуск), `←`/`→` (навигация)
- История решений текущей сессии с переходом по клику
- Фоновая очередь отправки статусов (PATCH в ActiveMap)
- Повторная отправка ошибочных заданий
- Кнопка «🔄 Обновить» — перезагрузить список без выхода из просмотра
- Форма отчёта об ошибках (🐛) с сохранением в `bug_reports.jsonl`
- Номер задания (#XXXXX) в заголовке карточки
- Версия приложения на экране входа
- Предупреждение при достижении лимита 500 заданий

### Исправлено

- Фотографии не загружались из-за некорректного определения поля-счётчика (`num_main_photo` — это индекс, а не количество)
- Кнопка «Пропустить» работала только один раз
- Задания со статусом «Завершено» попадали в список (добавлен фильтр по стадии)
- Кастомные поля показывали UUID-ключи и `[object Object]`
- Смена решения после успешной отправки не запускала повторный PATCH
